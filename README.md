# 숭실대학교 인근 주거 가성비 분석 및 적정 주거비 예측 서비스 (SSU Housing Analysis)

본 프로젝트는 숭실대학교 인근 대학가(동작구 및 관악구 일대)의 최근 3개년 주거용 부동산 실거래 데이터를 분석하여, 대학생들의 주거 비용(실질주거비) 부담 현황을 파악하고 머신러닝을 통해 합리적인 자취방 선택을 돕는 가성비 핫스팟 지도와 예측 서비스를 제공합니다.

---

## 📊 1. 사용된 데이터 정보 (Data Specification)

공공데이터포털의 **국토교통부 실거래가 오픈 API**에서 추출한 최근 3개년(2023~2025년) 데이터를 바탕으로 대학생 주거 현실에 맞게 정제된 데이터를 사용합니다.

*   **수집 대상 주택 유형**: **연립다세대 및 오피스텔** 
    *   *(단독/다가구 주택의 경우 지번 데이터 누락이 잦아 위치 특정이 불가하여 분석 대상에서 제외)*
*   **물리적 거리 제한**: 숭실대학교 중앙 기준 **직선거리 1.5km 이내** 매물만 포함
*   **경제적 타당성 제한**: 대학생들의 가용 예산을 고려하여 **보증금 1억 원 이하**의 월세 매물만 취급
*   **통합 지표 도입 (월 실질주거비)**: 보증금의 기회비용을 월세로 환산하여 전월세 가격을 단일 지표로 통합 비교합니다.
    *   `월 실질주거비 = 월 임대료(월세) + (보증금 * 5.5% / 12)`
    *   *전월세 전환율은 5.5%를 기준점으로 설정*

*(현재 정제 파이프라인을 통과한 최종 분석 데이터는 약 3,450여 건입니다.)*

## 🎯 2. 주요 제공 서비스 (Features)

본 프로젝트는 단순히 데이터를 보여주는 것을 넘어, 실질적인 솔루션을 제공하기 위해 **데이터 분석**과 **데이터 예측** 2가지 핵심 서비스를 제공합니다.

### A. 데이터 분석 영역: 가성비(저평가) 핫스팟 지도 시각화
머신러닝 모델(B)이 해당 매물의 스펙(면적, 연식 등)을 바탕으로 예측한 '적정 실질주거비'보다 **실제 거래된 실질주거비가 더 저렴한 매물(즉, 가성비 꿀매물)**을 찾아냅니다. 
이 가성비 매물들의 군집도를 계산하여 숭실대 인근 지도를 바탕으로 **히트맵(Heatmap) 핫스팟** 형태로 시각화합니다.
*   **효과**: 학생들은 방을 구하기 전, "어느 블록(구역)으로 가야 같은 조건 대비 월세가 저렴한지" 거시적인 타겟 지역을 설정할 수 있습니다.

### B. 데이터 예측 영역: 머신러닝 기반 적정 실질주거비 판독기
자취방을 알아볼 때 해당 매물이 '시세 대비 비싼지, 싼지' 객관적으로 판단할 수 있도록 돕는 예측 모델입니다.
- **예측 대상**: 입력된 조건(거리, 면적, 건축년도, 층)에 맞는 '월 실질주거비'의 적정가
- **사용 알고리즘**: 비선형적 관계 학습과 단조 제약(Monotonic Constraints) 적용에 뛰어난 `HistGradientBoostingRegressor` 사용
  - *단조 제약 조건(Monotonic Constraints) 적용 이유*: 단순 데이터 학습 시, 노이즈나 특이 케이스로 인해 "건축년도가 오래될수록 오히려 가격이 비싸게 예측"되는 등 인간의 상식에 위배되는 비선형적 과적합 문제가 발생했습니다. 이를 방지하고자 알고리즘을 교체하여 **"면적이 넓을수록 비싸짐(+)", "건축년도가 최신일수록 비싸짐(+)", "숭실대와 거리가 멀어질수록 저렴해짐(-)"** 이라는 상식적인 룰을 모델에 강제 주입하여 예측의 신뢰도를 극대화했습니다.
- **예측 방식**: `ml_train_data.csv` (약 3.4만 건의 단독다가구 포함 전체 데이터)를 기반으로 학습.
*   **효과**: 부동산에서 특정 매물을 본 직후, 계약 전 해당 방의 적정가를 모델을 통해 산출해봄으로써 정보 비대칭에 의한 불합리한 계약(호갱)을 예방할 수 있습니다.

## ⚠️ 3. 프로젝트의 한계 및 면책 조항 (Limitations)

본 프로젝트의 데이터 분석 및 예측 결과는 국토교통부에 신고된 **실거래가 건물 스펙(연식, 면적, 층수 등)**만을 기반으로 도출되었습니다. 
따라서 다음과 같은 현실적인 요소들은 반영되지 않았으므로 **참고용 지표**로만 활용해야 합니다.

1.  **관리비 및 부대비용 누락**: 월세 외에 매월 고정적으로 지출되는 관리비(인터넷, 수도, 전기 등) 데이터는 포함되어 있지 않습니다.
2.  **내부 옵션 및 리모델링 여부**: 채광(남향/북향), 풀옵션(세탁기, 에어컨 등) 구비 여부, 최근 내부 인테리어 리모델링 여부 등 가격에 큰 영향을 미치는 '정성적/내부적 가치'는 파악이 불가합니다.
3.  **단독/다가구 원룸 데이터의 부재**: 정확한 위치 맵핑을 위해 상세 지번이 누락된 단독/다가구 원룸 데이터가 배제되었으므로, 전체 대학가 원룸 시장의 100%를 대변하지는 않습니다.

## 🛠️ 4. 개발 환경 구축 및 실행 방법

### 1) Conda 가상환경 생성 및 필수 라이브러리 설치
```bash
conda create -n ssu-housing python=3.11 -y
conda activate ssu-housing
pip install -r requirements.txt
```

### 2) 서비스 실행 파이프라인
프로젝트 루트 디렉토리에서 아래 명령어를 실행하면, **데이터 전처리 -> 모델 학습 -> 핫스팟 지도 생성** 파이프라인이 일괄 가동됩니다.
```bash
python srcs/main.py
```
*   가동이 완료되면 `data/processed/hotspot_map.html` 파일이 생성되며, 브라우저에서 인터랙티브 지도를 확인할 수 있습니다.
*   또한 콘솔 창을 통해 내가 본 방의 정보를 입력하여 적정 월세를 예측해 볼 수 있는 인터페이스가 제공됩니다.
